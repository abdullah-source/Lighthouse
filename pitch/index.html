The issue we caught · fix this week

Rigor is the product.

In a real run, the parser counted "Brooks" and "Brooks Adrenaline GTS 23" as two different brands. The count split, the mention rate was understated.

The problem

LLMs name the same brand many ways: "Brooks," "Brooks Running," "Brooks Adrenaline." Naive aggregation treats them as different brands and the data lies.

The solution, this week

A normalization pass plus a canonical brand name. Every variant maps to one brand before we count. Then tests, so it can never silently return.

If you cannot resolve "Brooks" vs "Brooks Running," every number downstream is noise. This is the depth that earns the causal claim.
